### Day 2-3: Game Logic (6 hours saved)

#### Collision Detection (2 hours saved)

**My Prompt to Kiro:**
```
Create checkCollision function that checks if tetromino collides 
with board boundaries or existing pieces, handling pieces spawning 
above visible board (y < 0)
```

**What Kiro Generated:**
```typescript
export const checkCollision = (
  board: Board,
  piece: Tetromino,
  position: Position
): boolean => {
  for (let y = 0; y < piece.shape.length; y++) {
    for (let x = 0; x < piece.shape[y].length; x++) {
      if (piece.shape[y][x] !== 0) {
        const newX = position.x + x;
        const newY = position.y + y;
        
        if (newX < 0 || newX >= BOARD_WIDTH || newY >= BOARD_HEIGHT) {
          return true;
        }
        
        // CRITICAL: This check handles pieces spawning above board
        if (newY >= 0 && board[newY][newX] !== 0) {
          return true;
        }
      }
    }
  }
  return false;
};
```

**Critical Learning:** 
The `newY >= 0` check was an edge case I hadn't considered. Pieces spawn at negative Y values, and this prevents array index errors. **This one line saved me 2+ hours of debugging.**

**Time Saved:** 2 hours
**Lines Generated:** 25

#### Rotation Algorithm (1.5 hours saved)

**My Prompt:**
```
Create rotatePiece function that rotates tetromino 90 degrees clockwise 
using matrix transformation
```

**Kiro's Output:**
```typescript
export const rotatePiece = (piece: Tetromino): Tetromino => {
  const rotated = piece.shape[0].map((_, index) =>
    piece.shape.map(row => row[index]).reverse()
  );
  
  return { ...piece, shape: rotated };
};
```

**What I Learned:** 
Kiro used elegant functional programming (map + reverse) instead of nested loops. More readable and concise.

**Time Saved:** 1.5 hours

**Key Algorithm Sections:**

**1. Aggregate Height Calculation:**

```typescript
let totalHeight = 0;
for (let x = 0; x < BOARD_WIDTH; x++) {
  for (let y = 0; y < BOARD_HEIGHT; y++) {
    if (tempBoard[y][x] !== 0) {
      totalHeight += (BOARD_HEIGHT - y);
      break;
    }
  }

score -= totalHeight * 0.5;
```

**2. Holes Detection:**
```typescript
let holes = 0;
for (let x = 0; x < BOARD_WIDTH; x++) {
  let blockFound = false;
  for (let y = 0; y < BOARD_HEIGHT; y++) {
    if (tempBoard[y][x] !== 0) {
      blockFound = true;
    } else if (blockFound) {
      holes++; // Empty cell below filled cell = hole
    }
  }
}
score -= holes * 35;
```

## Key Learnings

### Prompt Engineering Lessons

**What Works:**
✅ Specific, detailed prompts with context
✅ Including edge cases explicitly
✅ Mentioning technology stack (TypeScript, React)
✅ Specifying return types and parameters
✅ Iterating on initial output

**Example of Good Prompt:**
```
Create [function name] that:
- Takes [parameters with types]
- Does [specific behavior]
- Handles [edge case]
- Returns [type]
- Use [technology/pattern]
```

**What Doesn't Work:**
❌ Vague requests ("make a game")
❌ No context or constraints
❌ Accepting first output without review
❌ Not testing immediately
